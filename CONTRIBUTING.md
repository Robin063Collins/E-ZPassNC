# Contribution Guidelines

Thank you for your interest in contributing to **E-ZPassNC**! We value community involvement and welcome contributions that help improve this project.

---

## How to Contribute

1. Fork the repository to your GitHub account.
2. Clone your forked repository to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes following the coding standards.
5. Commit your changes with a clear and descriptive message.
6. Push your branch to your fork.
7. Open a Pull Request to the main repository.

---

## Reporting Issues

If you encounter a bug or unexpected behavior:
- Check existing issues to avoid duplicates.
- Clearly describe the issue.
- Include steps to reproduce the problem.
- Add screenshots or logs if applicable.

---

## Suggesting Features

We welcome new ideas!
- Explain the feature clearly.
- Describe the problem it solves.
- Provide examples or mockups if possible.

---

## Code of Conduct

- Be respectful and inclusive.
- Provide constructive feedback.
- Avoid offensive or harmful language.
- Help create a welcoming environment for all contributors.

---

## Review Process

- All pull requests are reviewed by maintainers.
- Feedback may be provided for improvements.
- Once approved, the changes will be merged into the main branch.

Thank you for helping improve **E-ZPassNC**!
